# 需求分析与总体设计

## 需求分析
- 以《EduRisk-TabNet_plus.md》为唯一技术规格，实现完整流程：数据→缺失指示m→MAF→TabNet→DM-GFS→SASC→Head→DACOS→统一训练目标→主实验与消融→绘图与报告。
- 仓库为空，需建立标准工程骨架，提供配置、脚本、测试、结果与图表目录。
- 全部说明文字为中文，代码注释、docstring、日志信息也必须为中文。
- 提供合成数据生成脚本，包含缺失与K类有序标签，保证冒烟实验可运行。
- 按里程碑小步提交，每个里程碑完成后执行并汇报验证命令。

## 总体设计
- 数据层：合成数据与真实数据的读取、预处理、缺失指示m生成与缓存。
- 表征层：MAF、TabNet、DM-GFS、SASC、Head 组成的可组合模块，按配置启用。
- 目标层：DACOS 与统一训练目标，支持主实验与消融设置。
- 训练层：训练与评估管线、指标统计、日志与结果落盘。
- 绘图与报告层：从结果目录读取指标与曲线，输出图表与报告。

## 里程碑规划
1. 建立工程骨架与基础文档，明确目录职责与总体流程。
2. 提供最小可运行版本：合成数据、模型骨架、训练与评估主流程。
3. 完整实现所有模块与配置、主实验与消融流程。
4. 补充绘图脚本与报告生成，完善测试与可复现实验。
